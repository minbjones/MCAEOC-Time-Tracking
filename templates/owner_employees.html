Import Employees From CSV

Upload a UTF-8 CSV to create new employees and overwrite existing ones in one step. Existing employees are matched by payroll_id first, then email, then user_id. If a payroll_id does not already exist in the app database, the row is treated as a new employee and the database assigns the next payroll ID for that selected funding source using a 6-digit padded sequence. Each employee can be set to either Annual Leave or Personal Leave, not both. Imported rows reset the employee password to the CSV temporary password, or auto-generate a secure password if that field is blank. Owner rows must still include an explicit temporary password.

Required columns: first_name, last_name, email, hire_date, temporary_password, personal_leave or leave_type, funding_id or funding_source. Optional columns: payroll_id, user_id, role_name, workflow, is_active. Import also accepts the older department_id, department_name, and reports_to_user_id headers for compatibility.
